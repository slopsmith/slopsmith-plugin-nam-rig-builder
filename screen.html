Global FX layered around every tone of every song. Pre runs first (sees the raw DI — typical place for input gate / compressor / tuner). Post runs after the cab IR (typical place for limiter / room reverb / headphone EQ). Each piece supports the same Library / VST / Bypass / reorder controls as a song chain. Signal flow: guitar → master_pre → song → master_post → output.

How to use: (1) ＋ Add piece → pick a Rocksmith effect OR a VST plug-in. VST pieces are ready to use immediately — just hit 🎛 Edit VST to tweak its parameters. Rocksmith pieces start unassigned: click Assign… then pick a NAM/IR from your library or paste a VST plug-in path. (2) Master pieces are global — they don't appear on the Gear tab.
🎸 AMP must be on for master to apply. If you flip the AMP button mid-song, Rig Builder now auto-reloads the chain (including master pre/post) ~1 second later — no need to leave and re-enter. Disable with window.__rbAmpAutoApply = false if it ever misfires.

🚀 Quick start — how this plugin works

Three ordered steps. After you've done them once, the plugin self-heals new songs as you open them — you don't need to come back here unless your Rocksmith install changes.

  1. Extract from gears.psarc — populates the gear-map JSON, every gear's photo, and the cab IRs with mic-position metadata. Use the Extract everything card below.
  2. Scan library & auto-assign — walks every song and assigns captures from the curated 1-to-1 mappings only. Anything uncurated lands in pending for you to pick later. Tick the ⚠ Allow tone3000 fuzzy fallback box below if you want the plugin to auto-search tone3000 for uncurated gears instead.
  3. Refine in the Gear catalog — for each amp set 🎚 Variants (clean / crunch / dist), or assign a NAM/VST via 📚 Library. The Songs editor lets you tweak per-song mic position + gain variant + swap gear.

Re-extracting only refreshes the metadata (photos, IRs, gear map). Your curated picks in rs_to_real.json are preserved automatically. Songs already in the DB pick up the new mic positions when you next open them; or click Rescan all to apply everywhere at once.

Scan library & auto-assign captures

Walks every PSARC / sloppak in your library and, for each gear in each tone, resolves a NAM / IR / VST in this order: (1) reuse an assignment you've already made for that gear in another song, (2) your curated picks (default_captures.json + gain_variants), (3) a fresh tone3000 search + download — unless you turn that step off below.

                Chain preloader DEFAULT
            

            

                Pre-loads every tone of the song into one big chain at song start,
                then switches between tones by toggling bypass instead of reloading
                NAMs. Eliminates the tone-change spike
                completely. Steady-state memory is higher (every NAM stays
                loaded) but on M-series Macs the trade-off is clear-cut.
            

            

                Turn it off if you're on a weak x86 laptop and hear crackling, or if
                a specific song is misbehaving — playback then falls back to the
                cooperative load/unload path. Kill-switch in DevTools:
                window.__rbMegaChain = false.
